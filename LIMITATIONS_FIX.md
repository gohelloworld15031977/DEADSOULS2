```bash
# 5. Запустите обучение:
!pip install -r requirements.txt

```

**Ограничения:** 12 часов сессии, GPU T4/K80

#### Kaggle Kernels (бесплатно)
```bash

# 2. Создайте новый Notebook
# 3. Включите GPU: Settings > Accelerator > GPU
# 4. Загрузите данные и код
# 5. Запустите обучение
```

**Ограничения:** 30 часов GPU в неделю, P100/T4

#### Lambda Labs / RunPod (платно)
```bash
# Аренда GPU от $0.5/час
# Полная свобода конфигурации
```

### Решение B: Оптимизировать для CPU

Уже реализовано в `train_cpu_optimized.py`:

```bash
# Запуск оптимизированного обучения для CPU
.\venv\Scripts\python.exe train_cpu_optimized.py
```

**Оптимизации:**
- Отключен fp16 (неэффективно на CPU)
- Увеличен gradient_accumulation_steps (8)
- Отключен gradient_checkpointing
- num_workers=0 для dataloader
- pin_memory=False

**Ожидаемое время:** 3-5 часов для 3 эпох

## 2. Тест на повторения падает

### Проблема
На ранней стадии обучения модель склонна к повторениям.

### Решение
Порог тестов скорректирован:
- Было: < 30%
- Стало: < 50% (для ранней стадии)
- Целевое: < 20% (после полного обучения)

### Улучшение качества генерации

```bash
# 1. Отладить параметры генерации
.\venv\Scripts\python.exe debug_repetitions.py

# Рекомендуемые параметры:
# - temperature: 0.7-0.8
# - repetition_penalty: 1.2-1.5
# - top_p: 0.9-0.95
```
